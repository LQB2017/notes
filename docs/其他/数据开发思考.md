# 数据开发思考

**目标**：产出高质量、可直接使用的数据资产。

**全流程**：需求澄清 → 数据源分析 → 模型设计 → 数据获取 → 清洗加工 → 质量校验 → 结果落地 → 服务化应用。

---

### 1. 需求澄清（Why）
- 明确业务问题与数据场景
- 定义核心指标与交付口径

### 2. 数据源分析（What）
- 盘点可用数据源：业务库、日志、第三方接口、文件等
- 评估数据时效、完整性、权威性

### 3. 模型设计（How）
- 梳理实体与关系，构建逻辑模型
- 统一维度与指标命名规范，形成数据字典

### 4. 数据获取（Extract）
- 选择抽取策略：全量 / 增量 / 流式
- 建立稳定可靠的采集链路，记录元数据

### 5. 清洗加工（Transform）
- 标准化：类型转换、编码统一、异常处理
- 业务化：衍生字段、聚合口径、维度退化
- 性能优化：分区、索引、并行计算

### 6. 质量校验（Validate）
- 规则引擎：唯一性、非空、阈值、波动率
- 对比验证：与源系统、历史数据、业务台账交叉核对
- 形成质量报告，问题数据可回溯

### 7. 结果落地（Load）
- 选择存储：数仓分层（ODS/DWD/DWS/ADS）、数据湖、API
- 设定生命周期与权限策略，保障安全合规

### 8. 服务化应用（Deliver）
- 提供多形态服务：BI 报表、API 接口、数据沙箱、Kafka 流
- 建立运营监控：调用量、失败率、时效性
- 持续迭代：基于业务反馈与数据增长优化模型与链路

---

> 以终为始，让数据“可理解、可信任、可复用”，是数据开发的最高准则。
